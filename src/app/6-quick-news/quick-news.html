A round-up of the smaller Angular 22 news: tooling, accessibility, router, hydration and testing.

1. TypeScript 6 & Node

  • TypeScript 6.0 is supported (and becomes the minimum for new workspaces).
  • Minimum Node version bumped to 22 LTS - Node 20 support is dropped.
  • TS 6 highlights usable in Angular code: satisfies on config maps, const type parameters, smarter narrowing through aliased conditions.

2. @angular/aria is stable

The package graduates from experimental with 12 stable headless patterns: Toolbar, Tabs, Tree, Menu, Listbox, Combobox, Accordion, Radio Group, Dialog, Tooltip, Disclosure and Grid. You keep your own markup and styles; the directives bring the roles, keyboard navigation and focus management.

3. Trap: paramsInheritanceStrategy default change

The router default flips from 'emptyOnly' to 'always': child routes now inherit all parent route params and data. Watch out for name collisions - a parent :id now shows up in every child's params. 

// Restore the old behavior if needed:
provideRouter(routes, withRouterConfig({
  paramsInheritanceStrategy: 'emptyOnly',
}))

4. Incremental hydration is stable

With SSR, @defer blocks can now hydrate independently instead of hydrating the whole page at once - less JavaScript up front, faster interactivity. 

@defer (hydrate on viewport) {
  <app-reviews />
} @placeholder {
  <p>Loading reviews…</p>
}

5. TestBed.getLastFixture()

New testing helper: retrieve the most recently created component fixture, even when it was created indirectly (e.g. by a test harness or a router navigation in the test). 

await router.navigate(['/products/1'])
const fixture = TestBed.getLastFixture()
expect(fixture.nativeElement.textContent).toContain('Espresso')
